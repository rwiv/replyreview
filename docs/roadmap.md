# ReplyReview Roadmap

본 문서는 ReplyReview 데스크톱 애플리케이션의 장기적인 개발 로드맵입니다. 초기 구상된 리뷰 데이터 파싱 및 AI 기반 답글 생성 기능을 포함하는 MVP 버전을 시작으로, 안정적인 실행 파일 배포까지의 전체적인 마일스톤과 핵심 개발 트랙을 정의합니다.

## Phase 1: MVP 완성 (핵심 리뷰 답글 생성 기능)

### Track 1.1: 로컬 애플리케이션 기반 구조 및 설정 시스템 구축

이 트랙은 데스크톱 애플리케이션의 뼈대를 만들고 사용자의 민감한 정보(API 키)를 안전하게 관리하는 기반을 다집니다.

- **UI 프레임워크 구성:** PySide6 기반의 메인 윈도우 및 전체적인 사용자 인터페이스 레이아웃 설계
- **설정 관리 인프라:** 로컬 파일(`config.json`)을 활용한 설정 데이터 영속성 보장 및 상태 관리 모델 도입
- **API 설정 인터페이스:** 사용자로부터 외부 AI 서비스 연동을 위한 인증 키를 입력받고 관리하는 UI/UX 구현

### Track 1.2: 리뷰 데이터 입력 및 시각화 파이프라인 개발

사용자가 외부 쇼핑몰 리뷰 데이터를 손쉽게 앱으로 가져오고, 이를 직관적으로 확인할 수 있는 데이터 처리 흐름을 만듭니다.

- **데이터 파일 연동:** CSV 및 Excel 형식의 데이터를 앱으로 간편하게 가져오기 위한 드래그 앤 드롭 입력 인터페이스 구축
- **데이터 파싱 시스템:** 입력된 스프레드시트 파일을 해석하여 앱 내부에서 사용 가능한 리뷰 데이터 모델로 변환
- **리뷰 뷰어 UI:** 파싱된 리뷰 목록을 사용자가 읽고 선택하기 편한 카드 형태의 리스트 뷰로 시각화

### Track 1.3: AI 답글 생성 엔진 및 비동기 처리 로직 연동

외부 AI 모델을 활용하여 리뷰 맥락에 맞는 답글을 자동으로 생성하고, 사용자에게 끊김 없는 경험을 제공하는 핵심 기능을 완성합니다.

- **AI 외부 연동:** LangChain을 활용하여 개별 리뷰의 문맥을 분석하고 적절한 답글을 생성하는 프록시 시스템 도입
- **비동기 처리 아키텍처:** 네트워크 API 호출 시 UI 화면 멈춤(Freeze) 현상을 방지하기 위한 백그라운드 작업 인프라 구축
- **결과 상호작용 도구:** 생성된 답글을 화면에 표시하고, 원클릭으로 클립보드에 복사하여 외부 플랫폼에 활용할 수 있는 편의 기능 추가

## Phase 2: 앱 안정화 및 배포 파이프라인 구축

### Track 2.1: 오류 처리 시스템 및 사용자 피드백 강화

애플리케이션 사용 중 발생할 수 있는 다양한 예외 상황에 유연하게 대처하여 프로그램의 안정성과 사용자 신뢰도를 높입니다.

- **예외 상황 감지 인프라:** 지원하지 않는 파일 형식, 네트워크 통신 지연, API 인증 실패 등 주요 오류 케이스 감지 로직 구현
- **시각적 피드백 시스템:** 백그라운드에서 발생한 오류를 콘솔 에러가 아닌 사용자 친화적인 대화상자 형태의 메시지로 전달

### Track 2.2: 독립 실행 파일(Executable) 패키징 및 배포 체계 확립

개발이 완료된 애플리케이션을 일반 사용자가 별도의 개발 환경 설정 없이 즉시 실행할 수 있는 형태로 빌드합니다.

- **실행 파일 빌드 시스템:** PyInstaller를 활용하여 파이썬 인터프리터 및 의존성 라이브러리가 묶인 단일/디렉토리 형태의 실행 파일 생성 체계 구성
- **배포 설정 최적화:** 애플리케이션 고유 아이콘 적용 및 빌드 결과물의 안정성을 확보하기 위한 빌드 스펙(Spec) 고도화
