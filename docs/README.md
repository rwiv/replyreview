# Specification Documents (명세서 진입점)

이 문서는 AI 코딩 에이전트(Claude Code, Gemini CLI 등) 및 프로젝트 참여 개발자가 프로젝트의 전체적인 맥락과 명세서 구조를 파악하기 위한 진입점(Entry point)입니다. 모든 상세 구현에 앞서 이 디렉토리의 문서들을 우선적으로 참조해야 합니다.

## 1. 프로젝트 컨텍스트 요약

**ReplyReview** 프로젝트는 네이버 스마트스토어 리뷰 데이터를(CSV/Excel) 로드하여, LangChain(OpenAI)을 통해 맥락에 맞는 리뷰 답글을 자동 생성해주는 데스크톱 애플리케이션입니다.

- **핵심 목표**: 포트폴리오 목적으로 PySide6 기반의 수려한 GUI(카드 리스트 뷰, 드래그 앤 드롭)와 외부 AI 연동, PyInstaller 빌드 파이프라인을 보여주는 것.
- **주요 제약사항**: 
  - 외부 DB나 백엔드 서버 없이 로컬 데스크톱 앱으로 동작.
  - OpenAI API 키는 사용자가 앱 내 UI에서 입력하고 로컬의 `config.json`으로 저장 및 관리.
  - 답글 생성은 비동기로 처리되며 화면이 멈추지(Freeze) 않아야 함.

## 3. 에이전트 행동 지침

코딩 에이전트가 본 프로젝트의 코드를 작성하거나 수정할 때 다음 원칙을 준수해야 합니다.

1. **명세서 기반 구현**: 구현 중 모호한 부분이 있다면 반드시 `docs/features.md` 및 `docs/tech-spec.md`를 우선적으로 확인합니다. 명세서에 정의된 정책을 임의로 변경하지 마십시오.
2. **비동기 처리**: PySide6 환경에서 외부 API(LangChain)를 호출할 때는 `QThread` 또는 `asyncio` 등을 활용하여 메인 GUI 스레드가 블로킹되지 않도록 구현해야 합니다.
3. **오류 처리**: 파일 파싱 오류, API 키 미입력 오류, 네트워크 오류 등 발생 가능한 예외 상황은 반드시 콘솔 에러가 아닌 UI 상(예: QMessageBox 또는 텍스트 피드백)에 표시되어야 합니다.
