# Project Documentation

이 문서는 AI 코딩 에이전트(Claude Code, Gemini CLI 등) 및 프로젝트 참여 개발자가 프로젝트의 전체적인 맥락과 명세서 구조를 파악하기 위한 진입점(Entry point)입니다.

## 1. 프로젝트 개요

**ReplyReview** 프로젝트는 네이버 스마트스토어 리뷰 데이터를(CSV/Excel) 로드하여, LangChain(OpenAI)을 통해 맥락에 맞는 리뷰 답글을 자동 생성해주는 데스크톱 애플리케이션입니다.

**주요 제약사항**: 
- 외부 DB나 백엔드 서버 없이 로컬 데스크톱 앱으로 동작.
- OpenAI API 키는 사용자가 앱 내 UI에서 입력하고 로컬의 `config.json`으로 저장 및 관리.
- 답글 생성은 비동기로 처리되며 화면이 멈추지(Freeze) 않아야 함.


## 3. 주요 명령어

AI 에이전트 및 개발자가 로컬 환경에서 자주 사용하는 필수 명령어 목록입니다.

- **의존성 설치**: `uv sync`
- **애플리케이션 실행**: `uv run python -m replyreview`
- **테스트 실행 (전체)**: `uv run pytest`
- **코드 타입 체크**: `uv run pyright`
- **코드 포맷팅**: `uv run ruff format`
- **코드 린팅**: `uv run ruff check`

## 5. 개발자 행동 지침

본 프로젝트의 코드를 작성하거나 수정할 때 다음 원칙을 준수해야 합니다.

1. **명세서 기반 구현**: 구현 중 모호한 부분이 있다면 반드시 `docs/features.md` 및 `docs/tech-spec.md`를 우선적으로 확인합니다. 명세서에 정의된 정책을 임의로 변경하지 마십시오.
2. **비동기 처리**: PySide6 환경에서 외부 API(LangChain)를 호출할 때는 `QThread` 또는 `asyncio` 등을 활용하여 메인 GUI 스레드가 블로킹되지 않도록 구현해야 합니다.
3. **테스트 및 모킹**: 외부 AI API 의존성으로 인해 테스트가 복잡해지지 않도록, **Abstract Base Class(ABC)**를 기반으로 AI 클라이언트를 추상화합니다. 테스트 시에는 모킹 라이브러리 대신 `FakeAIClient`와 같은 가짜 객체를 사용하여 검증합니다.
4. **오류 처리**: 파일 파싱 오류, API 키 미입력 오류, 네트워크 오류 등 발생 가능한 예외 상황은 반드시 콘솔 에러가 아닌 UI 상(예: QMessageBox 또는 텍스트 피드백)에 표시되어야 합니다.
